# [TalleMalle] 프로젝트 요구사항 명세서 (SRS)

## 1. 프로젝트 개요
*   **프로젝트명:** TalleMalle (탈래말래)
*   **목적:** 택시비 절감을 위한 사용자 간 동승(Recruit) 매칭 및 전문 드라이버 배차/운행 관리 플랫폼.
*   **주요 기술 스택:**
    *   **Backend:** Java Spring Boot, Spring Security (OAuth2, JWT), JPA, MySQL, WebSocket (Stomp).
    *   **Frontend:** Vue 3 (Composition API), Pinia, Vite, TailwindCSS, Axios.
    *   **Infra:** Docker, Nginx (Nginx Proxy Manager), GitHub Actions (CI/CD).

---

## 2. 사용자 역할 (User Roles)

### 2.1 승객 (User/Passenger)
*   동승 모집글을 작성하거나 참여할 수 있는 일반 사용자.
*   결제 수단을 등록하여 운행 완료 후 결제를 수행함.

### 2.2 드라이버 (Driver)
*   배차 요청(Call)을 수락하고 승객을 운송하는 서비스 제공자.
*   운행 상태를 관리(수락, 운행 시작, 완료)하고 수익을 정산받음.

---

## 3. 기능적 요구사항 (Functional Requirements)

### 3.1 회원가입 및 인증 (Auth)
*   **이메일 회원가입:** 이메일 중복 확인, 닉네임 중복 확인, 비밀번호 암호화 저장.
*   **이메일 인증:** 가입 후 이메일로 발송된 UUID 링크를 클릭하여 계정 활성화.
*   **본인 인증:** PortOne(포트원) API를 통한 실명 및 본인 인증 필수.
*   **소셜 로그인:** Kakao, Google 등 외부 OAuth2 제공자를 통한 로그인 및 추가 정보(전화번호 등) 입력 프로세스.
*   **로그인/로그아웃:** JWT 기반 인증 (HttpOnly Cookie 사용), 로그아웃 시 쿠키 제거.
*   **비밀번호 관리:** 비밀번호 찾기(이메일 전송) 및 초기화 기능.

### 3.2 동승 모집 (Recruit/Ride-sharing)
*   **모집글 생성:** 출발지/목적지(위경도 포함), 출발 시간, 모집 인원, 메모 입력.
*   **지도 기반 검색:** 현재 지도 화면의 위도/경도(SouthWest, NorthEast) 범위를 기반으로 실시간 모집 현황 조회.
*   **모집 참여/퇴장:** 
    *   사용자는 하나의 모집에만 참여 가능.
    *   방장(Owner)이 퇴장할 경우 해당 모집글은 자동으로 삭제(취소)됨.
*   **상태 관리:** 모집 중(IDLE), 방장(OWNER), 참여자(JOINED) 상태에 따른 UI 분기.

### 3.3 실시간 채팅 (Chat)
*   **WebSocket(Stomp) 통신:** 동승 그룹 멤버 간 실시간 메시지 송수신.
*   **메시지 이력:** 이전 채팅 내역 무한 스크롤 조회.
*   **읽지 않은 메시지:** 채팅방별 읽지 않은 메시지 수 표시 및 알림.
*   **이미지 전송:** AWS S3(예상) Presigned URL을 통한 이미지 업로드 및 채팅 내 공유.
*   **멤버 관리:** 채팅 참여자 목록 확인 및 특정 사용자 차단 기능.

### 3.4 드라이버 전용 기능 (Driver)
*   **드라이버 인증:** 일반 사용자와 분리된 드라이버 전용 로그인/회원가입.
*   **배차(Call) 관리:**
    *   실시간 대기 중인 콜 목록 조회 및 상세 확인.
    *   콜 수락(Accept) -> 운행 시작(Driving) -> 운행 완료(Complete) -> 정산(Settlement) 단계별 상태 관리.
*   **수익 통계:** 일간/월간 운행 건수 및 총 수익 확인 위젯 제공.
*   **운행 이력:** 과거 수행한 콜 목록 및 정산 세부 내역 조회.

### 3.5 결제 시스템 (Payment)
*   **결제 수단 등록:** Toss Payments(토스페이먼츠) 빌링키 발급 및 등록.
*   **카드 관리:** 기본 결제 수단 설정 및 등록된 카드 삭제.
*   **자동 결제:** 운행 완료 시 드라이버의 요청에 의해 승객에게 등록된 빌링키로 요금 자동 청구.
*   **정산 관리:** 결제 완료 후 시스템 수수료를 제외한 드라이버 정산 로직.

### 3.6 알림 및 고객 지원 (Support & Misc)
*   **공지사항(Notice):** 관리자가 작성하는 공지사항 조회 및 상세 보기 (CRUD).
*   **FAQ:** 자주 묻는 질문 목록 제공.
*   **알림(Notification):** 동승 신청, 수락, 운행 시작 등 주요 이벤트 발생 시 실시간 알림 서비스.
*   **안심번호:** 사용자 개인정보 보호를 위한 안심번호 설정 기능.

## 4. 비기능적 요구사항 (Non-Functional Requirements)

### 4.1 보안 (Security)
*   **JWT 인증:** 모든 API 요청에 대해 JWT 토큰 검증 및 권한 체크 (승객/드라이버 구분).
*   **개인정보 보호:** 휴대전화 번호 및 결제 정보는 암호화 처리 또는 외부 보안 솔루션 이용.
*   **CORS 설정:** 허용된 도메인(프론트엔드)에서의 요청만 수락.

### 4.2 성능 및 가용성 (Performance & Availability)
*   **실시간성:** WebSocket 및 SSE를 활용하여 지연 없는 실시간 정보 동기화.
*   **이미지 최적화:** 채팅 이미지 업로드 시 S3 Presigned URL을 사용하여 서버 부하 분산.
*   **반응형 UI:** 모바일 기기 사용을 고려한 TailwindCSS 기반의 반응형 디자인 적용.

### 4.3 확장성 (Scalability)
*   **Dockerize:** 각 마이크로서비스(Backend, Frontend, DB)의 컨테이너화를 통한 배포 편의성 확보.
*   **Nginx 역방향 프록시:** 도메인 및 SSL 관리를 위한 Nginx 설정.

## 5. 화면 구조 (User Flow Summary)

1.  **로그인/가입:** 소셜/이메일 로그인 -> 본인 인증 -> 메인 이동.
2.  **동승 찾기:** 지도 탐색 -> 모집글 클릭 -> 상세 정보 확인 -> 참여 신청.
3.  **동승 중:** 채팅방 입장 -> 실시간 대화 -> 드라이버 배차 대기.
4.  **운행 및 결제:** 드라이버 도착 -> 운행 시작 -> 목적지 도착 -> 자동 결제 -> 리뷰 작성.
5.  **드라이버:** 대기 모드 -> 콜 수락 -> 승객 픽업 -> 운행 완료 -> 수익 확인.
